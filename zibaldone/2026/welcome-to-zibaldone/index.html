ワードクラウドプログラムとその仕組みについて（数学編）

Created on April 17, 2026

2026   ·   python   ·   Programming

https://github.com/Kenyu-f/Kenyu-f/blob/main/Python.py 　には、ワードクラウドと呼ばれるものを実装するためのpythonコードが公開されている。これを作成したのは私の学校の教員だ。今回は、この仕組みや背景を数学的な視点から解明しようと思う。

この記事は、私の数学の理解度にかなり依存してしまっている（私の場合、代数に偏りすぎているが、おおよそ学部2年までの数学は理解可能である）。だが、この分野ではほとんど無知なので、あなたとはほぼ知識レベルが変わらないと予想される。であるから、分からないと思った用語にはリンクをつけたり、補足情報をつけようと思う。

一番手っ取り早い方法はAIに聞くことだと思ったので、十分に数学的な教育がされたChatGPTを用いて、質問をしてみた。次のように返ってきた。

入力は有限文字列 $S = c_1 c_2 \dots c_T$（各 $c_i$ は文字集合 $\Sigma$ の要素）とする。ここでの目的は、この文字列を「単語」と呼ばれる連続部分列の列に分解することである。まず「列分割」を厳密に定義する。整数列 $0 = t_0 < t_1 < \dots < t_n = T$ を区切り位置（boundary）と呼び、このとき各区間に対応する部分列 $w_i = c_{t_{i-1}+1} \dots c_{t_i}$ を単語と定義する。このとき分割は $W = (w_1, \dots, w_n)$ と表される。したがって問題は、与えられた $S$ に対して適切な区切り位置列 $(t_0, \dots, t_n)$ を選ぶことに等しい。長さ $T$ の文字列に対しては、各隣接文字の間で切るか否かの2択があるため、可能な分割の総数は $2^{T-1}$ であり、全探索は指数時間を要するため実用的ではない。

そこで「どの分割が自然か」を評価するために確率を導入する。単語列 $W$ に対して確率 $P(W)$ を定義し、これが最大となる分割を求める問題として定式化する。このとき $\arg\max$ という記号を用いるが、これは「与えられた関数の値を最大にする引数（ここでは $W$）を返す演算」を意味する。すなわち $\arg\max_W P(W)$ は「$P(W)$ を最大にするような $W$」を表す。したがって問題は $W^* = \arg\max_W P(W)$ となる。しかし $P(W)$ をそのまま扱うことは困難であるため、確率モデルに構造を仮定する。ここで用いるのがマルコフ性である。マルコフ過程とは「将来の状態が現在の状態のみに依存し、過去の履歴には依存しない」という性質を持つ確率過程である。この仮定を単語列に適用すると、各単語は直前の単語にのみ依存するとして \(P(W) \approx \prod_{i=1}^n P(w_i \mid w_{i-1})\) と分解できる。さらに計算の都合上、対数をとる。対数関数は単調増加であるため、最大化問題は保存され、 \(\arg\max_W P(W) = \arg\max_W \log P(W)\) が成り立つ。すると積は和に変換され、 \(\log P(W) = \sum_{i=1}^n \log P(w_i \mid w_{i-1})\) となる。ここで負号をつけると最小化問題に変換でき、 \(\arg\max_W P(W) = \arg\min_W \sum_{i=1}^n -\log P(w_i \mid w_{i-1})\) となる。この $-\log P$ を「コスト」と呼ぶ。したがって問題は「コストの総和が最小になる分割を求める問題」に帰着する。

Janomeのような形態素解析器では、単語だけでなく品詞も扱うため、各単語は $(\text{表層形}, \text{品詞})$ の組として扱われる。さらに各単語には単独での出現のしやすさを表すコスト $\text{cost}(w_i)$ と、隣接する単語間の接続の自然さを表すコスト $\text{cost}(w_i, w_{i+1})$ が与えられる。したがって全体のコストは \(\sum_i \text{cost}(w_i) + \text{cost}(w_i, w_{i+1})\) と書ける。次に、すべての候補分割を効率的に扱うためにラティス（lattice）と呼ばれる構造を導入する。ラティスとは有向非巡回グラフ（DAG）であり、ノードは「位置区間と単語候補の組」すなわち $(i, j, w)$ で表され、これは部分列 $c_i \dots c_j$ が単語 $w$ に対応することを意味する。エッジは「連続する区間」の間に張られ、すなわちノード $(i, j)$ と $(j+1, k)$ の間に有向辺が存在する。これにより、ある始点（文頭）から終点（文末）までの任意の経路は、1つの分割に対応する。したがって問題は「このグラフ上でコストが最小となる経路を求める問題」に変換される。

この最短経路問題は、グラフが非巡回であるため動的計画法で解くことができる。このとき用いられるのがViterbiアルゴリズムである。これは各位置までの最小コストを逐次的に計算する方法であり、状態 $dp(j)$ を「位置 $j$ まで到達する最小コスト」と定義する。更新式は \(dp(j) = \min_i \{ dp(i) + \text{cost}(i \to j) \}\) であり、ここで $\text{cost}(i \to j)$ は区間 $(i, j)$ に対応する単語のコストと接続コストを含む。初期条件は $dp(0) = 0$ とする。すべての位置についてこの更新を行うことで、最終位置 $T$ における $dp(T)$ が最小コストとなり、そのときの経路を逆に辿ることで最適な単語列が復元される。

以上をまとめると、形態素解析とは、文字列を部分列に分割する組合せ最適化問題であり、それを確率モデル（マルコフ仮定）によってコスト最小化問題に変換し、さらにラティスというグラフ構造上の最短経路問題として解いている。すなわち「離散的な分割の選択問題」を「連続的なスコア（コスト）の最適化問題」に写像している点に本質がある。

ここで、分からなかった言葉や概念をまとめてみよう。私の場合、「単語列 (W) に対して確率 (P(W)) を定義し」、「$\arg\max_W P(W)$」、確率過程、マルコフ過程、形態素、「隣接する単語間の接続の自然さを表すコスト」、DAG、動的計画法などである。
